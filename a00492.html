◆ NUMQWORDS

#define NUMQWORDS   400U

Number of Samples to store of coil voltage.

Number of samples, ~2MHz, 100kHz res. freq = 20 cycles.

For calculating Q using resonant decay, ADC/DMA will capture NUMQWORDS worth of data to be post processed for peak coil voltage information. These samples are stored on the stack so significant RAM space must be available.

◆ NUMPEAKS

#define NUMPEAKS   40U

Number of peaks to collect/search for averaging Q.

Max number of coil voltage peaks to try to collect while searching through NUMQWORDS worth of sample points. This should be quite a bit larger than NUM_Q_AVG.

◆ NUM_Q_AVG

#define NUM_Q_AVG   6U

Average Q over 'NUM_Q_AVG' readings.

For improved Q readings, Q can be averaged over a number of cycles. Must be able to collect enough peaks to average over though. Depending on resistance in the system, coil voltage signal could decay significantly and if this average is too large, the last few samples could be corrupted.

◆ NUM_RES_FREQ_AVG

#define NUM_RES_FREQ_AVG   10U

Average frequency over 'NUM_RES_FREQ_AVG' cycles.

Greater than 2 for eliminating initial glitches.

Number of AC cycles to average resonant frequency over. Must be less than then number of expected AC cycles from measureQ() span.
