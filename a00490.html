◆ NUM_COILS

#define NUM_COILS   3U

Set the number of coils in application.

Qi Stack currently supports applications with 1-4 coils but FW can easily scale to accommodate more coils. If application is a single coil implementation, NUM_COILS = 1U, all coil selection FW is removed from application.

◆ EXT_RAIL_CNTRL

#define EXT_RAIL_CNTRL

Includes APIs for Front-end DC-DC voltage controller.

If defined, modifies Qi stack to include APIs that handle external rail voltage control in function such as, PID, state-machine, and fault handler. This should be defined if application is a rail control transmitter topology. All APIs are linked using qi_ext_drv.h.

◆ USB_PD_CNTRL

#define USB_PD_CNTRL

Enables FW components related to USB-PD front-end transmitters.

If defined, certain FW components are enabled for supporting UBD-PD front-end transmitters such as MP-A22. Functions that switch PDO source, check PRx guaranteed power, PID controller, etc. are enabled. External function can be defined in qi_ext_drv.h.

◆ LOW_POWER_PING

#define LOW_POWER_PING

Sets Qi sleep mode state, expecting application to call Sleep().

If defined, Qi stack will set sleep mode state after each coil sweep when there isn't an object found on the interface surface. This can be used to place application into sleep which greatly reduces the standby power of the transmitter. The sleep duration is set by the devices sleep watchdog timing #pragma config SWDTPS = PS512. If not defined, a delay of 500ms will be inserted between each coil sweep and the Qi sleep state flag will not be set.

Note: Application is responsible for adding condition to enter/exit sleep mode using Qi sleep APIs.

◆ USE_VIRTUAL_DDM_VCOIL

#define USE_VIRTUAL_DDM_VCOIL

Enables virtual PPS for two DDM channels.

If defined, (PWM4 must be free), stack switches out two external DDM I/O pins (coil connection) for virtual I/O pins using remappable PWM. This frees up two device pins for low pin count applications.

◆ SMP_INV_RAIL_VOLTAGE

#define SMP_INV_RAIL_VOLTAGE

Enables qi stack to sample inverter voltage.

If defined, stack enables mechanism to sample inverter voltage at the same rate as the inverter input current. This should only be defined in applications without the front-end DC-DC converter. In these topologies, the inverter voltage is being sampled (regulated) by controller and the DC-DC is responsible for passing the measured output voltage.
