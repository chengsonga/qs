Detailed Description

DMA SFR mapping / APIs that scale with peripheral instances defined in qi_project_peripheral_defines.h. The DMA source registers scale with ADC channel mapping automatically. The destination registers are fixed: Q-data is on the stack and used with Q routines, coilVoltageDMABuffer[] and coilCurrDMABuffer[] variables for capturing peak data.

void Drv_DMA_Init ( void  )

Initialize DMA module.

Function: InitDMA Description: 1 DMA channel is used to move coil voltage data when calculating quality factor. This allows over sampling coil voltage while still processing the 4SWBB control loop A second DMA channel is required for capturing coil voltage data for DDM, this comes from feedback signal not Q feedback

Parameters
void
Returns
void

Initializes the DMA module for use by the Qi stack. DMA RAM address range is configured for all of RAM as Q-measurement uses Stack and address range may vary.

One DMA channel is initialized for Q-measurement. This channel captures NUMQWORDS worth of data when trigger is set within Q routines. If COIL_VOLT_DDM_PK or
COIL_CURR_DDM_PK is defined, additional DMA channels are enabled for capturing four sample points every switching cycle.
